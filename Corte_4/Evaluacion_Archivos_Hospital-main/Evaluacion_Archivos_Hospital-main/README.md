# Evaluación de Organizaciones de Archivos en un Sistema Hospitalario

## Descripción del proyecto

Este proyecto tiene como objetivo desarrollar un sistema experimental en Python para almacenar, procesar y analizar información hospitalaria utilizando diferentes organizaciones de archivos: CSV y JSON.

El sistema permitirá comparar ambos formatos mediante métricas como tiempo de lectura, tiempo de escritura, tamaño del archivo, facilidad de procesamiento, facilidad de visualización y flexibilidad para almacenar información compleja.

## Escenario

Un hospital desea modernizar el manejo de información de pacientes y necesita decidir qué tipo de organización de archivos es más conveniente para almacenar y procesar sus datos.

La información simulada incluirá:

- Registros de pacientes.
- Consultas médicas.
- Signos vitales.
- Historial clínico.
- Medicamentos recetados.
- Citas médicas.
- Especialidades médicas.

## Fases del proyecto

### Fase I. Generación de datos

Se generarán conjuntos de datos simulados con:

- 1 millón de registros.
- 10 millones de registros.
- 20 millones de registros.

### Fase II. Implementación del sistema

El sistema permitirá:

- Leer archivos CSV y JSON.
- Agregar nuevos pacientes.
- Buscar pacientes por ID o nombre.
- Generar reportes médicos.
- Mostrar estadísticas básicas.

### Fase III. Experimentación

Se medirán las siguientes métricas:

- Tiempo de lectura.
- Tiempo de escritura.
- Tamaño del archivo.
- Facilidad de procesamiento.
- Facilidad de visualización.
- Flexibilidad para almacenar información compleja.

### Fase IV. Visualización de datos

Se generarán gráficas de:

- Barras.
- Líneas.
- Circular o pastel.
- Histogramas.

### Fase V. Evaluación del sistema

Se evaluará qué organización de archivos es más eficiente, flexible, fácil de analizar y recomendable para un sistema hospitalario real.

### Fase VI. Análisis de datos

Se realizará análisis descriptivo mediante:

- Promedios.
- Máximos.
- Mínimos.
- Frecuencias.
- Interpretación de tendencias y patrones.
